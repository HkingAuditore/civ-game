# 政治系统 - 完整机制

> **核心逻辑**: `config/politicalStances.js`, `logic/officials/manager.js`

---

## 1. 政治立场

### 1.1 立场光谱

| 光谱 | 代表立场 | 核心诉求 |
|-----|---------|---------|
| left | 原始公社、马克思主义、社民 | 平等、集体、改革 |
| center | 共和、儒家、道家 | 法治、传统、自由 |
| right | 部落长老、贵族寡头、法家 | 等级、效率、传统 |

### 1.2 条件类型 (`politicalStances.js` CONDITION_TYPES)

每个立场有独特的满足条件：

| 条件类型 | 检测内容 |
|---------|---------|
| `stratum_approval_above/below` | 阶层满意度 |
| `stratum_influence_above` | 阶层影响力占比 |
| `stratum_living_standard` | 阶层生活水平等级 |
| `stability_above/below` | 国家稳定度 |
| `avg_tax_above/below` | 平均税率 |
| `coalition_includes` | 执政联盟成员 |
| `at_war` / `at_peace` | 战争状态 |
| `resource_price_above/below` | 资源物价 |

### 1.3 立场效果

**满足条件时** (`activeEffects`):
```javascript
{
    stability: +0.05,
    approval: { peasant: +4, worker: +4 },
    tradeBonus: +0.12,
    industryBonus: +0.18,
}
```

**不满足时** (`unsatisfiedPenalty`):
```javascript
{
    stability: -0.02,
    approval: { peasant: -2, worker: -2 },
}
```

## 2. 执政联盟

### 2.1 联盟组成

执政联盟由影响力最高的阶层组成（通常2-3个）。

### 2.2 联盟效果

- 联盟成员满意度影响稳定度权重更高
- 某些政治立场要求特定阶层在联盟内

---

## 3. 内阁协同

### 3.1 协同度计算 (`cabinetSynergy.js`)

```javascript
synergy = 基于官员政治立场分布计算

// 等级
if (synergy >= 0.8): "高度协同"
if (synergy >= 0.5): "基本协同"
if (synergy >= 0.3): "松散协同"
else: "内斗严重"
```

### 3.2 主导派系

当填充率≥50%时，最多的政治光谱成为主导派系：

| 主导 | 效果 |
|-----|-----|
| left | 价格管制生效、福利政策可用 |
| center | 平衡发展加成 |
| right | 效率加成、税收优化 |

### 3.3 协同效果

高协同度带来：
- 政策执行效率↑
- 稳定度加成
- 内部冲突↓

低协同度导致：
- 效率降低
- 派系斗争
- 政变风险↑

## 4. 稳定度

### 4.1 影响因素

| 因素 | 影响 |
|-----|-----|
| 联盟满意度 | 正相关 |
| 阶层冲突 | 负相关 |
| 战争状态 | 负相关 |
| 内阁协同 | 正相关 |
| 官员忠诚 | 低忠诚负相关 |

### 4.2 稳定度效果

| 稳定度 | 效果 |
|-------|------|
| >70 | 生产+5%，无负面 |
| 50-70 | 正常 |
| 35-50 | 宣战概率↑，效率↓ |
| <35 | 叛乱风险↑，AI激进 |

## 5. 法令系统

### 5.1 法令类型

- **临时法令**: 有持续时间，到期失效
- **永久政策**: 需要条件维持

### 5.2 法令效果来源

法令效果通过 `getActiveDecreeEffects()` 聚合应用到游戏各系统。

## 6. 玩法策略

### 6.1 官员招募

- 选择与现有官员政治立场相近的新官员
- 避免极端对立立场同时存在
- 关注立场条件是否能满足

### 6.2 联盟经营

- 提升联盟成员阶层满意度
- 确保关键阶层在联盟内
- 利用政策调整联盟边界

### 6.3 内阁管理

- 维持50%以上填充率以激活主导派系
- 左派主导解锁价格管制
- 右派主导提高税收效率

### 6.4 稳定度维护

- 避免同时得罪多个大阶层
- 战时注意稳定度下降
- 低稳定度时减少激进政策
