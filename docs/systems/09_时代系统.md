## 2. 升级要求

### 2.1 要求类型

| 字段 | 说明 |
|-----|-----|
| `minPopulation` | 最低人口 |
| `minBuildings` | 最低建筑总数 |
| `techsRequired` | 需要的科技列表 |
| `minStability` | 最低稳定度 |

### 2.2 示例（青铜→铁器）

```javascript
requirements: {
    minPopulation: 50,
    minBuildings: 15,
    techsRequired: ['bronze_working', 'wheel'],
    minStability: 40
}
```

cost: { science: 科技点, silver: 银币, culture: 文化 }

## 4. 时代加成

各时代提供的全局加成：

| 时代 | gatherBonus | industryBonus | militaryBonus |
|-----|-------------|---------------|---------------|
| 石器 | +20% | - | - |
| 青铜 | +40% | +20% | +20% |
| 铁器 | +60% | +40% | +40% |
| 古典 | +80% | +60% | +60% |
| 中世纪 | +100% | +80% | +80% |
| 文艺复兴 | +120% | +100% | +100% |
| 工业 | +150% | +150% | +120% |

其他可能的加成：
- `cultureBonus`: 文化产出
- `taxBonus`: 税收效率
- `populationGrowth`: 人口增长
- `stabilityBonus`: 稳定度基础值

## 5. 时代影响

### 5.1 解锁内容

- 科技：该时代及之前的科技可研究
- 建筑：`epoch ≤ 当前时代`的建筑可建造
- 兵种：对应时代的军事单位
- 外交：更高级的条约和机制

### 5.2 兵种淘汰

老时代兵种在新时代效率下降：
```javascript
if (currentEpoch - unitEpoch > obsoleteAfterEpochs):
    effectiveness × 0.25
```

---

## 6. 玩法策略

### 6.1 升级时机

- 不要急于升级，确保满足所有条件
- 稳定度不足时先稳定内政
- 准备好升级成本资源

### 6.2 跨时代优势

- 新时代加成大幅提升生产
- 新兵种对旧兵种有显著优势
- 新建筑效率更高

### 6.3 落后惩罚

- 旧时代兵种被克制
- 无法使用新建筑
- AI可能拉开差距
