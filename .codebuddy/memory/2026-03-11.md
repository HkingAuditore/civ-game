## Zustand Store 状态管理迁移（渐进式）

### 完成的工作：
1. **安装 Zustand v5.0.11** 作为状态管理库
2. **创建 11 个领域 store** (`src/stores/`目录)：
   - `useUIStore` - 标签页、游戏速度、暂停、存档设置、日志等 UI 状态
   - `useResourceStore` - resources, market, priceHistory, taxes, fiscalActual 等
   - `usePopulationStore` - population, classWealth, classApproval, stability 等 ~30 个字段
   - `useBuildingStore` - buildings, buildingUpgrades, techsUnlocked, epoch, daysElapsed
   - `useEconomyStore` - eventEffectSettings, priceControls, festivalEffects 等
   - `useMilitaryStore` - army, militaryCorps, generals, activeFronts, activeBattles 等
   - `useDiplomacyStore` - nations, diplomaticReputation, overseasInvestments 等
   - `usePoliticsStore` - rebellionStates, rulingCoalition, decrees, promiseTasks 等
   - `useOfficialStore` - officials, ministerAssignments, officialCapacity 等
   - `useEventStore` - currentEvent, eventHistory, achievements 等
   - `useTradeStore` - merchantState, tradeRoutes, tradeStats
3. **创建工具层**：
   - `storeUtils.js` - getStoreSnapshot/applyStoreSnapshot/batchStoreUpdate（存档/加载兼容）
   - `useStoreSync.js` - useGameState → Zustand 单向同步 hook（核心桥接层）
   - `useStoreBridge.js` - useStoreField/useStoreFields（组件迁移辅助工具）
4. **在 App.jsx 集成** `useStoreSync(gameState)` 实时同步

### 迁移策略说明：
- **渐进式**：useGameState（3787行）保持不变，通过同步层将 useState 值实时推到 Zustand stores
- **零破坏**：useGameLoop（6963行）和 useGameActions（6609行）完全不用修改
- **新组件直接用 store**：可以 `import { useUIStore } from '../stores'` 精细订阅
- **后续可逐步替换 useState**：将 useGameState 内的 useState 逐步替换为 store 读写
