# 需求文档：外资利润税控制（含逐国覆盖）

## 引言

当前外资企业面板（国际经济概览 → 外资企业 Tab）展示了各国在玩家国内的投资情况，包括纳税额、利润流出和实际税率。但玩家**无法主动调节**对本国外资企业的利润税率——税率完全由外交关系（条约/共同体/附庸状态）自动决定。

本功能在外资企业面板中增加一个**玩家可控的利润税率政策**，允许玩家设定全局默认税率档位，并可对特定国家单独覆盖。该税率变更会产生外交后果（影响国家关系）和经济后果（影响外资投资意愿和撤资概率）。

此外，玩家还可以在**外交详情面板**（NationDetailView → 贸易Tab → "该国在我国的投资"区域）中直接调节对该国的税率档位，实现逐国精细化管理。

### 现有系统背景

- **外交规则税率**（基础税率）：由 `processForeignInvestments` 中的外交关系逻辑决定（共同体10%、条约25%、附庸25%、无条约60%）
- **`FOREIGN_INVESTMENT_POLICIES`**：已存在 `normal`/`increased_tax`/`heavy_tax` 三档配置（含 `taxRate` 和 `relationImpact`），当前已扩展为四档（含 `preferential`）
- **状态**：`foreignInvestmentPolicy` 状态和 `set_foreign_investment_policy` action 已存在
- **UI**：`ForeignCapitalTab` 已渲染全局税率档位选择控件

### 数据结构扩展

```js
// 现有（全局统一）
foreignInvestmentPolicy: 'normal'

// 扩展后（全局 + 逐国覆盖）
foreignInvestmentPolicy: 'normal',                // 全局默认
foreignInvestmentPolicyOverrides: {               // 逐国覆盖（可选）
    'nation_id_1': 'preferential',   // 对该国优惠
    'nation_id_2': 'heavy_tax',      // 对该国重税
}
```

## 需求

### 需求 1：全局税率控制UI（已实现，需调整）

**用户故事：** 作为一名玩家，我希望在外资企业面板中设定全局默认的利润税率档位，作为所有外资国家的基准政策。

#### 验收标准

1. WHEN 玩家打开国际经济概览面板并切换到"外资企业"Tab THEN 系统 SHALL 在面板顶部显示全局税率档位选择控件。
2. WHEN 玩家查看税率档位选项 THEN 系统 SHALL 显示四个档位（优惠5%、正常10%、加税25%、重税50%），每个档位仅显示名称和税率百分比，**不显示关系影响数值**。
3. WHEN 玩家切换全局税率档位 THEN 系统 SHALL 更新 `foreignInvestmentPolicy` 状态，对所有**未单独设置覆盖**的国家生效。
4. IF 某国已有逐国覆盖设置 THEN 全局档位变更 SHALL NOT 影响该国的有效税率。

### 需求 2：逐国税率覆盖（外资企业面板入口）

**用户故事：** 作为一名玩家，我希望在外资企业面板中对特定国家单独设置不同于全局默认的税率档位，以便对不同国家采取差异化的税收策略。

#### 验收标准

1. WHEN 玩家查看外资企业面板中某国的投资分组头部 THEN 系统 SHALL 在该国分组头部显示一个税率档位选择器（如下拉菜单或小按钮组），默认显示"跟随全局"。
2. WHEN 玩家为某国选择了非"跟随全局"的档位 THEN 系统 SHALL 将该国ID和选择的档位存入 `foreignInvestmentPolicyOverrides`，并在下一个模拟tick对该国的外资企业生效。
3. WHEN 玩家将某国的档位切回"跟随全局" THEN 系统 SHALL 从 `foreignInvestmentPolicyOverrides` 中移除该国的条目。
4. WHEN 某国有逐国覆盖时 THEN 系统 SHALL 在该国分组头部的"实际税率"显示中反映覆盖后的有效税率，并以视觉标记（如不同颜色或图标）区分于跟随全局的国家。

### 需求 3：逐国税率覆盖（外交详情面板入口）

**用户故事：** 作为一名玩家，我希望在外交详情面板中查看某国信息时，也能直接调节对该国的外资利润税率，无需切换到国际经济概览面板。

#### 验收标准

1. WHEN 玩家在外交详情面板（NationDetailView）的"贸易"Tab中查看"该国在我国的投资"区域 THEN 系统 SHALL 在该区域的头部显示一个税率档位选择按钮（与外资企业面板中的逐国选择器功能一致）。
2. WHEN 玩家通过外交详情面板修改某国的税率档位 THEN 系统 SHALL 同步更新 `foreignInvestmentPolicyOverrides`，效果与在外资企业面板中修改完全一致。
3. IF 该国在玩家国内没有投资 THEN 系统 SHALL 不显示税率选择按钮（或以禁用状态显示）。
4. WHEN 该国有逐国覆盖设置时 THEN 系统 SHALL 在按钮上显示当前覆盖的档位名称，而非"跟随全局"。

### 需求 4：利润税率档位设计

**用户故事：** 作为一名玩家，我希望有多个清晰的税率档位可选，以便在税收收入和外交关系之间做出权衡。

#### 验收标准

1. WHEN 系统提供税率档位选项 THEN 系统 SHALL 包含以下预设档位（复用并扩展现有 `FOREIGN_INVESTMENT_POLICIES`）：
   - **优惠税率**（~5%）：吸引外资，关系+2/月（隐式生效，不在UI显示）
   - **正常税率**（~10%）：默认档位，无关系影响
   - **加税**（~25%）：增加税收，关系-5/月（隐式生效，不在UI显示）
   - **重税**（~50%）：大幅增加税收，关系-15/月（隐式生效，不在UI显示）
2. WHEN 逐国覆盖生效时 THEN 系统 SHALL 对该国使用覆盖档位的税率和关系影响，而非全局默认档位。
3. IF 玩家选择的税率档位与外交规则税率冲突 THEN 系统 SHALL 取两者中的较高值作为最终有效税率。

### 需求 5：利润税对国家关系的影响（后台逻辑）

**用户故事：** 作为一名玩家，我希望调整利润税率会隐式影响与外资来源国的关系，作为政策选择的自然后果。

#### 验收标准

1. WHEN 玩家设定了非"正常"的税率档位（全局或逐国） THEN 系统 SHALL 在每个模拟周期对相应国家施加对应的关系变化。
2. WHEN 逐国覆盖存在时 THEN 系统 SHALL 对该国使用覆盖档位的关系影响值，而非全局默认值。
3. WHEN 关系变化生效时 THEN 系统 SHALL 仅影响在本国实际拥有外资企业的国家。
4. WHEN 关系因税率变化而降低 THEN 系统 SHALL 在外交日志中记录原因。
5. 关系影响数值（+2、-5、-15等）**不在UI中显示**，仅作为后台逻辑隐式生效。

### 需求 6：利润税对投资意愿的影响

**用户故事：** 作为一名玩家，我希望高税率能抑制新的外资流入并加速现有外资撤离，以便通过税率政策间接控制外资规模。

#### 验收标准

1. WHEN 系统评估AI国家对玩家国的投资意愿时 THEN 系统 SHALL 使用该国的有效税率档位（优先逐国覆盖，否则全局默认）来调整投资概率。
2. WHEN 玩家设定高税率档位（加税或重税） THEN 系统 SHALL 降低AI国家对玩家国的新投资概率。
3. WHEN 玩家设定优惠税率 THEN 系统 SHALL 提高AI国家对玩家国的新投资概率。
4. IF 税率变更导致某外资企业从盈利变为亏损 THEN 系统 SHALL 触发现有的连续亏损撤资机制。

### 需求 7：UI交互与信息展示

**用户故事：** 作为一名玩家，我希望税率控制的UI简洁直观，在多个入口保持一致的操作体验。

#### 验收标准

1. WHEN 玩家查看外资企业面板中某国分组头部 THEN 系统 SHALL 显示该国的有效税率（综合外交规则和玩家设定）。
2. WHEN 某国有逐国覆盖时 THEN 系统 SHALL 在该国分组头部以视觉标记区分（如显示覆盖档位名称或不同颜色）。
3. WHEN 玩家在外交详情面板和外资企业面板中修改同一国家的税率 THEN 两处 SHALL 保持状态同步。
4. WHEN 玩家选择非默认税率（全局或逐国） THEN 系统 SHALL 以醒目颜色标注当前处于非默认状态。

## 边界情况与技术考量

1. **存档兼容性**：旧存档中不存在 `foreignInvestmentPolicyOverrides`，系统应默认为空对象 `{}`，所有国家跟随全局默认。
2. **税率叠加规则**：有效税率 = `Math.max(外交规则税率, 有效档位税率)`，其中有效档位 = 逐国覆盖 ?? 全局默认。
3. **性能**：逐国覆盖仅存储有差异的国家，不存储"跟随全局"的国家，保持数据稀疏。
4. **关系影响频率**：每月结算一次关系影响，逐国覆盖的国家使用各自档位的关系影响值。
5. **状态传递**：`foreignInvestmentPolicyOverrides` 需要通过 `useGameState` → `App.jsx` → `DiplomacyTab` → `DiplomacyLayout` → `NationDetailView` → `ForeignInvestmentFromNation` 完整传递。同时需要传递 `onPolicyOverrideChange` 回调。
6. **逐国选择器设计**：建议使用紧凑的下拉菜单或小按钮组，选项为"跟随全局"+"四个档位"共五个选项，避免占用过多空间。
