# 需求文档：资源详情 - 产业链 UI 重构

## 引言

当前资源详情模态框中的「产业链」标签页存在两个核心问题：

1. **英文名称泄露**：部分建筑在产业链 UI 中显示英文 ID（如 `smelter`、`forge`、`steel_mill`），而非中文名称。根源是 `BuildingChip` 组件使用 `BUILDINGS.find(b => b.id === bid)` 查找建筑定义时，如果产业链配置中引用的建筑 ID 在 `buildings.js` 中不存在对应条目（即该 ID 未注册为正式建筑），组件就会 fallback 显示原始英文 ID。

2. **布局与设想差距大**：当前产业链以「垂直列表卡片」形式呈现（每条产业链一个卡片，内含上游/下游列表），但用户期望的是**横向流程图式布局**——以节点（圆圈）代表资源/阶段，箭头表示流向，形成 **从左到右的产业链流程图**，当产业链出现分叉时向下分支，直观展示上下游关系和产业链拓扑结构。

本需求旨在修复英文名问题并将产业链 UI 重构为流程图式布局。

## 需求

### 需求 1：修复建筑英文名显示问题

**用户故事：** 作为一名玩家，我希望产业链 UI 中所有建筑名称都显示中文，以便我能清晰理解每个环节涉及的建筑。

#### 验收标准

1. WHEN 产业链 UI 渲染建筑标签 THEN 系统 SHALL 通过 `BUILDINGS.find(b => b.id === bid)` 查找建筑定义，若找到则显示 `bDef.name`（中文名）
2. IF 建筑 ID 在 `BUILDINGS` 配置中不存在 THEN 系统 SHALL 显示一个友好的 fallback 名称（如去掉下划线并首字母大写的格式化名称，或标记为「未知建筑」），而非直接显示原始英文 ID
3. WHEN 产业链配置（`industryChains.js`）中引用了不存在于 `buildings.js` 中的建筑 ID THEN 开发者 SHALL 审查并修正这些不一致的引用（确保产业链中的建筑 ID 与 `buildings.js` 中已定义的建筑 ID 一一对应）
4. WHEN 所有修复完成 THEN 产业链 UI 中 SHALL 不再出现任何英文建筑 ID 作为显示名称

### 需求 2：产业链布局重构为横向流程图

**用户故事：** 作为一名玩家，我希望产业链以流程图的形式呈现，以便我能直观地看到资源从上游到下游的完整生产链路和分叉关系。

#### 验收标准

1. WHEN 用户打开产业链标签页 THEN 系统 SHALL 以**横向流程图**的形式渲染产业链，每个阶段/资源用节点表示，阶段之间用箭头连接，流向为从左到右
2. WHEN 产业链存在分叉（一个阶段产出被多个下游消耗） THEN 系统 SHALL 将分叉路径纵向排列，箭头从分叉点分别指向各下游节点
3. WHEN 产业链存在合并（多个上游汇入同一阶段） THEN 系统 SHALL 将多条上游箭头汇聚到该阶段节点
4. IF 当前资源是该产业链中的某个节点 THEN 系统 SHALL 用高亮样式（如金色边框/光晕）突出标记该节点，使用户一眼识别当前资源在链中的位置
5. WHEN 横向内容超出容器宽度 THEN 系统 SHALL 支持水平滚动，使用户能浏览完整链路
6. IF 某个阶段尚未解锁（不在当前时代范围内） THEN 系统 SHALL 以半透明/灰色样式渲染该节点，并显示解锁时代提示

### 需求 3：流程图节点内容设计

**用户故事：** 作为一名玩家，我希望流程图中的每个节点能展示关键信息（资源图标、阶段名称、关联建筑），以便我快速了解每个环节的详情。

#### 验收标准

1. WHEN 渲染资源节点 THEN 系统 SHALL 在节点中显示：资源图标、资源名称
2. WHEN 渲染阶段节点 THEN 系统 SHALL 在节点中显示：阶段名称、关联建筑列表（使用中文名称）、产出资源图标
3. WHEN 渲染连接箭头 THEN 系统 SHALL 使箭头样式清晰可辨（带方向箭头的线段或渐变线），颜色与项目现有 ancient-gold 主题一致
4. IF 节点包含的建筑已建造 THEN 系统 SHALL 在建筑名称旁显示已建造数量
5. IF 节点包含的建筑未建造 THEN 系统 SHALL 以灰色/低饱和度样式显示该建筑

### 需求 4：多条产业链的组织方式

**用户故事：** 作为一名玩家，当某个资源涉及多条产业链时，我希望能清晰地区分和浏览每条产业链的流程图。

#### 验收标准

1. WHEN 资源涉及多条产业链 THEN 系统 SHALL 为每条产业链分别渲染一个独立的流程图，每个流程图有清晰的产业链名称标题
2. WHEN 多条产业链流程图需要纵向排列 THEN 系统 SHALL 在每条链之间有足够的间距，并以产业链名称和角色标签（原材料/中间品/产出品）作为分隔
3. IF 资源不在任何已定义的产业链中 THEN 系统 SHALL 使用 fallback 逻辑，用建筑的 input/output 关系推断并展示简化的生产/消费关系图

### 需求 5：交互与响应式体验

**用户故事：** 作为一名玩家，我希望产业链流程图在不同窗口大小下都能良好显示，并且有流畅的交互体验。

#### 验收标准

1. WHEN 流程图在较窄的容器中显示 THEN 系统 SHALL 支持横向滚动，且有滚动指示器或提示
2. WHEN 用户悬停在某个节点上 THEN 系统 SHALL 通过轻微的视觉反馈（如亮度提升或边框高亮）指示交互性
3. WHEN 产业链数据加载完成 THEN 系统 SHALL 无明显的布局抖动，流程图平滑渲染

## 技术约束与边界条件

- **不引入新依赖**：流程图使用纯 CSS + JSX 实现（flexbox/grid 布局 + CSS 绘制的连接线/箭头），不引入 D3.js 或其他图表库
- **兼容现有组件风格**：使用项目现有的 ancient-gold/ancient-ink/ancient-parchment 颜色主题
- **性能考虑**：产业链数据是静态配置，节点数量有限（通常每条链5-12个阶段），不需要虚拟化
- **组件作用域**：所有改动限定在 `ResourceDetailModal.jsx` 中的产业链相关组件（`DynamicChainView`, `ChainCard`, `StageRow` 等），以及必要时修正 `industryChains.js` 中的配置不一致
- **现有系统复用**：复用现有的 `RESOURCES`、`BUILDINGS`、`STRATA` 配置查找逻辑，复用 `Icon` 组件，不创建新的数据模型
