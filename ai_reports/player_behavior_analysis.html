👥 玩家行为聚类

基于50名深度玩家的行为特征(建筑/科研/军事/外交/税收)，识别出以下玩家类型：

🏗 建设型 (Builder)
约占深度玩家 35%

以建筑购买为核心行为，建筑种类丰富(40-97种)，总建筑数千到万级。科研和军事参与度中等。代表玩家：709eec(97种建筑/2679次购买)、f6de6d(46种/16698次)。

特征：建筑种类>30，建筑操作占总操作>50%，军事参与度<30%
⚔️ 军事型 (Warmonger)
约占深度玩家 25%

军事操作占主导，频繁招募/解散部队。代表玩家：6f4ca5(军事9466次,建筑202次)、8104b5(军事6962次)、e5abd9(军事2353次)。

特征：军事操作>建筑操作，频繁解散部队(说明军费负担导致频繁裁军)
🤝 外交型 (Diplomat)
约占深度玩家 20%

外交操作极为频繁，部分玩家外交操作数甚至达到万级。代表玩家：f8de88(外交17915次)、88272(外交7700次)、b69616(外交4562次)。

特征：外交操作>1000次，部分玩家外交操作超过其他所有操作总和。说明外交系统有足够深度吸引特定玩家。
💰 政策型 (Economist)
约占深度玩家 20%

以税收调整和经济管理为核心。频繁调整税率，关注财政收支平衡。代表玩家：7c8ea8(税收768次)、c3909d(税收304次)、4d5666(税收285次)。

特征：税收操作>100次，有体系地研究科技、建造建筑配合经济发展
分群关键洞察
发现1：大量深度玩家(约40%)的 max_epoch 为 null，说明他们从未成功升级时代。这些玩家往往通过高倍速获得了巨量游戏天数(10-60万天)，但在时代进阶系统中完全停滞。可能是游戏速度过快导致资源/人口增长跟不上。
发现2：科研参与度极低——50名深度玩家中约60%的 total_research 为 0。科技系统可能存在引导缺失或成本过高的问题。
发现3：军事型玩家中，解散(Disband)事件数量极高(90881次,远超招募的27964次)。这意味着大量玩家在反复招募→财政崩溃→解散的循环中，军事系统的费用模型可能需要优化。

💹 市场物价趋势
物价核心发现
工具价格崩溃：tools 价格从初始71降至后期28-30，跌幅超过58%。这是当前最严重的价格异常——工具作为关键中间品，价格持续下跌可能导致整个制造业利润坍塌。
石材/布匹温和上涨：stone 从0.76升至1.75，cloth 从1.24升至2.29，呈健康的需求驱动增长。说明这两种资源的供需关系相对平衡。
食物价格稳定：food 价格长期稳定在1.0-1.3区间，说明农业系统基本平衡。
铁矿微降：iron 从35.8微降至36.0，整体稳定。但绝对值远高于其他资源(约30x)，可能造成铁矿相关建筑"暴利"。

🎯 优化建议
P0 紧急修复首次体验流失

数据依据：75%新玩家的会话<5分钟，D1留存仅7%。
根因分析：①游戏开局没有明确目标引导 ②科技系统被70%玩家忽略 ③时代升级条件模糊
具体建议：

  • 增加"首30分钟引导任务链"：建第一座农场→研究第一个科技→升级到100人口→进入青铜时代
  • 在科技面板增加"推荐科技"高亮标记，并在状态栏增加"下一步建议"提示
  • 首次打开时的教程弹窗应在3步以内讲清核心循环：建筑→资源→人口→时代
  • 考虑将初始资源增加30-50%，让新玩家更快感受到正反馈
P0 紧急解决后期通缩恶化

数据依据：CPI 从104.7降至55.4(跌幅47%)，tools价格跌幅58%。
根因分析：工具(tools)过度生产导致价格崩溃，拖累整体CPI
具体建议：

  • 工具需求模型升级：让更多高级建筑消耗tools(维护/升级)，增加tools需求端
  • 加强库存损耗：当前2%/tick的损耗率不够，建议对tools这类工业品提高到3-5%
  • 引入通缩自动修正：当CPI低于80时，自动增加消费倾向(如"便宜就多买"的市场行为)
  • 后期科技消耗tools：信息时代的科技研究应该需要大量tools作为原料
P1 高优化军事经济平衡

数据依据：解散/招募比3.25:1，后期单位数量达千万级。
根因分析：①军队维护费过高迫使频繁裁军 ②后期军事规模膨胀无上限
具体建议：

  • 军费占GDP比例软上限：军费超过GDP 30%时，维护费递增(而非线性)
  • 后期单位人口上限：信息时代单位应受人口/科技约束，避免无限招募
  • 降低前中期维护费：民兵/弓箭手等基础单位维护费降低20-30%
  • 增加"预备役"机制：允许低成本维持部分部队待命状态
P1 高修复科技系统参与度

数据依据：70%玩家从未研究过科技，深度玩家中60%科研为0。
根因分析：①科技面板可见性低 ②科技效果不直观 ③缺乏"为什么要研究"的引导
具体建议：

  • 科技引导弹窗：当科学点积累到第一个科技成本的50%时，弹出提示
  • 科技效果可视化：研究完成后用明显动画展示效果(如"+10%农场产量")
  • 自动研究选项：提供"自动研究下一个推荐科技"的便利功能
  • 降低前3个科技成本30%：让新玩家更快体验到科研的好处
P1 高优化时代7→8节奏断裂

数据依据：时代7→8平均耗时从17649天跳增到30210天(+71%)，标准差36115。
具体建议：

  • 审查冷战时代升级条件：降低科技/人口门槛约15-20%
  • 增加中期里程碑奖励：在每个时代中期提供"阶段性奖励"，减少枯燥感
  • 引入"时代进度条"提示：让玩家清楚知道距离下一时代的差距和路径
P2 中减少叛乱频率 / 优化叛乱体验

数据依据：伐木工/农民叛乱各2000+次，失业者叛乱影响28%玩家。
具体建议：

  • 失业者的brewing阶段门槛提高(需要更高组织度才能进入)
  • 叛乱发生后增加"冷却期"(同一阶层60-90天内不再重复叛乱)
  • 增加"怀柔政策"快捷操作，让玩家可以一键安抚即将叛乱的阶层
P2 中提升理念系统参与度

数据依据：54%的理念涌现被跳过。
具体建议：

  • 理念涌现时暂停游戏(防止玩家不注意而跳过)
  • 在理念选择界面增加"效果对比"预览
  • 增加"理念组合效应"，多个互补理念同时装备有额外加成
P2 中住房建筑策略多样化

数据依据：住房建筑占总建筑购买的35%，hut单项占20%。
具体建议：

  • 增加住房建筑的递增成本，防止无限堆叠
  • 增加其他提升人口上限的途径(公共设施、市政效果等)
  • 住房过多时引入"城市拥挤"debuff降低满意度
P3 低修复 Worker / React 错误

数据依据：889次Worker问题，215次React无限循环。
具体建议：

  • 排查React Error #185的来源组件，可能存在useEffect依赖循环
  • "window is not defined"说明Worker内引用了浏览器API，需要隔离
  • 大存档模拟跳过问题仍需进一步裁剪Worker数据payload
