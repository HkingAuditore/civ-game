# 官员产业政策系统策划案

> **设计理念**：在"哈耶克的文明"中，官员不是你的工具——他们是有自主意志的政治实体。产业政策的选择，本质上是国家与官僚集团之间"权力-利益"博弈的制度化表达。

---

## 一、问题诊断：当前系统的缺陷

### 1.1 玩家缺乏战略选择
当前系统中，官员**自动**用私人财富投资建筑（`processOfficialInvestment()`），玩家对此**完全没有控制权**。这意味着：
- 你无法阻止贪婪的官员疯狂扩张产业
- 你无法选择"高薪养廉"这种历史上常见的制度
- 官员置办产业对国家的影响是单一维度的——只有"好处"（增加建筑）和"隐患"（官员变富变强），没有真正的**博弈张力**

### 1.2 产权模型未充分利用
`ownerTypes.js` 中已经定义了 `STATE`（国有）类型，但从未被使用。这意味着游戏缺少了一个重要的经济制度维度：**国有企业/官营经济**。

### 1.3 官员属性与经济行为脱节
官员有完整的属性系统（`administrative`、`greed`、`level`），但这些属性对产业经营的影响仅体现在"投资决策概率"上。**行政能力高的官员和低的官员经营同一栋建筑，产出完全一样**——这浪费了属性系统的游戏深度。

---

## 二、核心设计：三种产业政策

### 设计哲学
每种政策都是一种**制度性妥协**——你解决了一个问题，必然引入另一个。这种"两难选择"才是策略游戏的核心乐趣。

```
         ┌─────────────────────────────────────────────┐
         │            官员产业政策（全局）                │
         │                                             │
         │   ┌───────────┐ ┌───────────┐ ┌───────────┐ │
         │   │  私产制    │ │ 高薪养廉  │ │ 代经营制  │ │
         │   │ (Private)  │ │(HighPay)  │ │ (State)   │ │
         │   │            │ │           │ │           │ │
         │   │ 官员自由   │ │ 禁止产业  │ │ 官营代管  │ │
         │   │ 投资经营   │ │ 高薪补偿  │ │ 国有分红  │ │
         │   └───────────┘ └───────────┘ └───────────┘ │
         └─────────────────────────────────────────────┘
```

### 2.1 🏛️ 私产制（Private）— "放养权贵"

> *"让他们富起来，他们就会忠心——直到他们比你还富。"*

**机制**：保持当前默认行为不变。官员用个人财富自主投资建筑，利润全部归官员所有，产权类型为 `OFFICIAL`。

#### 核心参数
| 参数 | 值 | 说明 |
|---|---|---|
| 薪资倍率 | 1.0x | 标准薪资 |
| 投资行为 | ✅ 允许 | 沿用现有 processOfficialInvestment() |
| 产权类型 | OFFICIAL | 官员私产 |
| 利润归属 | 100% 归官员 | 现有逻辑不变 |
| 基础腐败 | 0（通过 FINANCIAL_STATUS 自然产生） | 财务不满时腐败 |

#### 优势
- **忠诚度正循环**：财务满意的官员更忠诚（`financialSatisfied: +0.30/日`）
- **零国库负担**：不需要额外支出养官员
- **自动建设**：官员主动投资 = 免费的建筑产能
- **政治稳定**：右派阶层/官员满意度高

#### 风险
- **权臣风险**：富有的官员影响力暴涨（`calculateOfficialAbsoluteInfluence()` 中财富和产业数量是主要因子）
- **处置困难**：处置拥有大量产业的官员时，资产处理复杂，政变风险高（`COUP_WEALTH_THRESHOLD: 30000`, `COUP_PROPERTY_THRESHOLD: 3`）
- **利益固化**：右派官员疯狂扩张，左派官员几乎不投资（马克思主义投资概率=0），导致经济被右派集团绑架
- **国有份额萎缩**：官员产业越多，阶层产业占比越低，业主岗位减少

#### 适用场景
- 早期国库紧张时，借官员之手扩张经济
- 右派意识形态主导的政权
- 玩家愿意接受"放权→搏增长"的高风险策略

---

### 2.2 💰 高薪养廉（High Salary）— "花钱买干净"

> *"付得起廉洁的代价，才配谈廉洁。"*

**机制**：彻底禁止官员投资任何产业。作为补偿，官员薪资自动提升至 **1.6 倍**。官员不积累产业财富，忠诚度主要受薪资满足度驱动。

#### 核心参数
| 参数 | 值 | 说明 |
|---|---|---|
| 薪资倍率 | 1.6x | 基础薪资 × 1.6 |
| 投资行为 | ❌ 禁止 | 完全跳过投资/升级逻辑 |
| 产权类型 | 无 | 官员不持有产业 |
| 利润归属 | 不适用 | 无产业 |
| 腐败修正 | -2% | 全局腐败降低（反映制度性廉洁） |

#### 优势
- **官员透明**：无产业 = 无私利 → 官员更"纯粹"地为你服务
- **处置安全**：处置官员无需考虑资产问题，政变风险大降（因为没有 `COUP_WEALTH_THRESHOLD` 和 `COUP_PROPERTY_THRESHOLD` 的触发条件）
- **降低腐败**：全局腐败率下降 2%
- **影响力可控**：没有产业的官员，`calculateOfficialAbsoluteInfluence()` 中的 `wealthScore`、`propertyCountScore` 等全部为 0，影响力天然受限
- **左派满意**：社会主义/左派官员和阶层满意度提升

#### 风险
- **国库出血**：薪资支出增加 60%，对小国/穷国是沉重负担
- **失去自动建设**：官员不再帮你投资建筑，所有基础建设全靠国库
- **右派不满**：右派官员忠诚度下降（他们的"政治诉求"包含经济自由）；资本家、商人、地主阶层满意度下降
- **财务脆弱性**：官员唯一收入是薪资，一旦国库发不出工资（`salaryUnpaid: -0.30/日`），忠诚度崩溃极快

#### 适用场景
- 国库充裕、经济稳定时
- 左派/社会主义政权
- 需要频繁调换/处置官员时
- 玩家不信任官员的自主行为

#### 数值平衡思路
为什么是 1.6x 而不是 2.0x？

**数学依据**：
- 当前系统中，一个中等能力的官员，其产业年收入约等于其年薪的 0.5-1.5 倍
- 即：官员总收入 ≈ 薪资 + 产业收入 ≈ 薪资 × (1.5 ~ 2.5)
- 高薪养廉只补偿薪资部分的 0.6x（即 1→1.6），官员实际总收入**下降**
- 这是**故意的**：高薪养廉不是让官员过得更好，而是"用够用的钱换取他们放弃商业权利"
- 官员会**不满但可忍受**——如果你薪资再高到2.0x+，就没有博弈了

### 2.3 🏗️ 代经营制（State Managed）— "国有民营"

> *"产业是国家的，经营是你的——赚多赚少，看你本事。"*

**机制**：官员可以被"指派"经营建筑，但产业所有权属于国家（`STATE` 类型）。投资资金来自**国库**。利润大部分归国库，官员按自身**行政能力**获得**管理费分成**。官员的属性**直接决定**产业经营效率。

这是三种模式中**机制最丰富、博弈最深**的一种。

#### 核心参数
| 参数 | 值 | 说明 |
|---|---|---|
| 薪资倍率 | 1.0x | 标准薪资（管理费另算） |
| 投资行为 | ✅ 国库出资 | 投资决策仍基于官员profile，但扣国库的钱 |
| 产权类型 | STATE | 国有，附带 `managedBy: officialId` |
| 利润归属 | 国库为主 | 国库 = 利润 ×（1 - 管理费 - 腐败损耗） |
| 基础腐败 | +2% | 代理人问题天然存在 |

#### 关键公式：官员属性如何影响经营

这是整个系统的**核心创新点**——让官员的 stats 真正产生可感知的经济效果。

```
经营效率 = 基础效率 × (1 + 行政加成 + 等级加成)

  行政加成 = official.stats.administrative / 500
    → administrative=50 时 +10%，=100 时 +20%
    → 范围: 0% ~ +20%

  等级加成 = official.level × 0.02
    → level=5 时 +10%，level=10 时 +20%
    → 范围: 0% ~ +20%（假设最高等级10）

  总效率加成上限: +40%（行政+等级叠满）
```

```
管理费分成 = min(0.25, 0.10 + 行政加成 + official.level × 0.01)

  → 最低 10%（新手官员），最高 25%（顶级官员）
  → 高能力官员赚更多管理费 = 动机对齐
```

```
腐败损耗 = official.greed × 0.05

  → greed=0.5 时损耗 2.5%
  → greed=1.5 时损耗 7.5%
  → greed=3.0 时损耗 15%（极端贪婪）
  → 这部分利润"蒸发"——既不归国库也不归官员
```

```
国库实际收入 = 产业利润 × 经营效率 × (1 - 管理费 - 腐败损耗)

举例：一个 administrative=70, level=5, greed=1.0 的官员经营工厂：
  经营效率 = 1 + 0.14 + 0.10 = 1.24 → 产出提升 24%
  管理费 = min(0.25, 0.10 + 0.14 + 0.05) = 0.25 (达到上限)
  腐败损耗 = 1.0 × 0.05 = 0.05 (5%)
  国库获得 = 利润 × 1.24 × (1 - 0.25 - 0.05) = 利润 × 86.8%
  官员获得 = 利润 × 1.24 × 0.25 = 利润 × 31%
  蒸发部分 = 利润 × 1.24 × 0.05 = 利润 × 6.2%
```

#### 优势
- **属性终于有用了**：高行政能力的官员 = 更高效的代理人 → 玩家有动力培养和选拔特定属性的官员
- **国库增收**：产业利润主要流入国库，且高能力官员还能提升产出
- **处置灵活**：处置官员后，`STATE` 产权的建筑**不会消失**，只需要指派新的代管人
- **制度感**：模拟了真实的"官营经济"——国有企业由官僚代为经营，有效率也有腐败

#### 风险
- **国库出资压力**：投资成本从官员转移到国库
- **代理人问题**：贪婪的官员会"吃回扣"——腐败损耗是隐性的，玩家只能通过对比"理论利润"和"实际利润"来感知
- **低能力灾难**：如果指派了一个 administrative=20, greed=2.5 的官员：
  - 效率 = 1.04（几乎无加成）
  - 腐败 = 12.5%
  - 管理费 = 15%
  - 国库只能拿到利润的 75.4%，还不如阶层自己经营
- **投资决策仍受政治立场影响**：马克思主义官员不太会主动建议"国库投资"（他们本质上反对资本积累）

#### 适用场景
- 有高行政能力官员团队时
- 想要国家直接控制经济命脉
- 中央集权型政权
- 需要频繁调换官员但不想损失产业

## 三、政策切换：制度变革的代价

> *"每一次制度改革，都是一次政治地震。"*

### 3.1 切换冷却
- **冷却期**：90 天（约 1/4 年）
- **存储**：`lastPolicyChangeDay` 字段
- **UI 提示**：冷却中显示剩余天数，按钮灰色不可点击

### 3.2 切换矩阵

| 从 → 到 | 私产制 → 高薪养廉 | 私产制 → 代经营制 | 高薪养廉 → 私产制 | 高薪养廉 → 代经营制 | 代经营制 → 私产制 | 代经营制 → 高薪养廉 |
|---|---|---|---|---|---|---|
| **存量处理** | 没收官员产业 | 产权转为 STATE | 无存量 | 无存量 | 产权转为 OFFICIAL | 移除代管关系 |
| **补偿方式** | 2选1：补偿50%/强制没收 | 自动转换，官员变代管人 | — | — | 产业按购入价"赠予"官员 | 国库收回产业 |
| **忠诚度影响** | 右派官员 -15~-30 | 右派官员 -5~-10 | 左派官员 -10 | — | — | 左派官员 -5 |
| **阶层满意度** | 资本家/商人/地主 -10 | 资本家 -5 | 工人/农民 -5 | — | 工人/农民 -10 | 资本家 -5 |
| **难度** | ⭐⭐⭐ 高（最复杂） | ⭐⭐ 中 | ⭐ 低 | ⭐ 低 | ⭐⭐ 中 | ⭐⭐ 中 |

### 3.3 重点场景：私产制 → 高薪养廉（国有化没收）

这是最戏剧性的切换，弹出**确认弹窗**：

## 四、联动系统

### 4.1 忠诚度联动

不同政策下，忠诚度影响因子的**权重**不同：

```
                  私产制          高薪养廉        代经营制
                  ──────          ────────        ────────
财务满意度权重      1.5x            0.5x           1.0x
薪资满意度权重      1.0x            2.0x           1.0x
行政能力权重        1.0x            1.0x           1.5x
```

**设计意图**：
- 私产制下，官员的忠诚度主要取决于他们赚不赚到钱 → 财务因素放大
- 高薪养廉下，官员没有"自己赚钱"的渠道，全靠你发工资 → 薪资因素放大（发不出工资忠诚度崩盘极快）
- 代经营制下，高行政能力的官员能拿到更多管理费 → 行政因素放大（能力强的官员在这种制度下更忠诚）

### 4.2 处置系统联动

| 处置行为 | 私产制 | 高薪养廉 | 代经营制 |
|---|---|---|---|
| 解雇 | 官员带走所有私产 → 建筑计数减少 | 直接清退，无资产问题 | 官员离开，产业归国家，需指派新代管人 |
| 流放 | 私产50%没收，50%带走 | 直接流放 | 产业不受影响 |
| 处死 | 私产全部没收归国库 | 直接处死 | 产业不受影响 |
| 政变判定 | 财富+产业达标即可能政变 | 政变概率极低（无财富/产业基础） | 中等风险（有管理费收入但无产权） |

**核心差异**：代经营制下处置官员的代价最低——产业是国家的，换人即可。

### 4.3 内阁派系联动

- **左派主导内阁**：高薪养廉/代经营制的忠诚度惩罚减半
- **右派主导内阁**：私产制的投资概率 +20%（左派主导时 -50%，沿用现有的 `factionMod`）
- **意识形态冲突**：如果当前意识形态是"古典自由主义"却推行高薪养廉，右派阶层满意度额外 -15

### 4.4 科技/时代联动

可考虑以下科技解锁对应政策或提供政策加成（本次不实现，预留扩展接口）：

| 科技 | 效果 |
|---|---|
| 官僚制 | 解锁代经营制 |
| 法治 | 高薪养廉腐败降低再 -1% |
| 会计学 | 代经营制腐败损耗 -25%（玩家能"审计"了） |
| 国有化 | 代经营制效率加成 +5% |

## 七、博弈矩阵：为什么这三种模式有意义？

### 7.1 三角博弈

```
                     私产制
                    /      \
                   /        \
             控制力    vs    增长力
                 /            \
                /              \
          高薪养廉 ————————— 代经营制
                   效率 vs 国库
```

| 维度 | 私产制 | 高薪养廉 | 代经营制 |
|---|---|---|---|
| **经济增长** | ⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ |
| **国库收入** | ⭐ | ⭐⭐ (省腐败) | ⭐⭐⭐⭐ |
| **玩家控制** | ⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| **官员忠诚** | ⭐⭐⭐ | ⭐⭐⭐ (看国库) | ⭐⭐⭐ |
| **政变风险** | ⭐⭐⭐⭐⭐ | ⭐ | ⭐⭐ |
| **制度复杂度** | ⭐ | ⭐ | ⭐⭐⭐⭐ |
| **左派满意** | ⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| **右派满意** | ⭐⭐⭐⭐⭐ | ⭐ | ⭐⭐ |

### 7.2 典型玩家策略

1. **"放养开局"**：早期用私产制，让官员帮你建设经济 → 中期国库充裕后切换到高薪养廉或代经营制
2. **"铁腕集权"**：一开始就代经营制，亲自控制所有经济 → 需要招高行政力的官员
3. **"养肥了杀"**：私产制让官员积累产业 → 找借口处死 → 没收全部资产充公 → 再招新的
4. **"制度震荡"**：根据局势在三种模式间切换 → 但每次切换都有政治代价和冷却期

## 九、性能与存档兼容

### 性能
- 高薪养廉模式**减少**计算量（跳过投资/升级逻辑）
- 政策状态是全局单值，不增加 per-official 开销
- 代经营制的效率修正在 `actualProfitCache` 之后应用（乘法修正），不影响缓存

### 存档兼容
- 旧存档加载时 `officialPropertyPolicy` 默认为 `'private'`，完全向后兼容
- `managedProperties` 字段不存在时视为空数组
- 迁移函数自动填充缺失字段

## 十、未来扩展方向（本次不实现）

1. **个体政策**：允许对单个官员设置不同的产业政策（如：大部分人高薪养廉，但允许特定能臣代经营）
2. **产业配额制**：限制官员最多持有 N 处产业
3. **审计系统**：代经营制下，花费行政资源审计官员可以发现腐败并追回损失
4. **官员派系**：官员之间形成利益集团，集体抵制政策变更
5. **科技解锁**：特定科技解锁对应政策模式或提供加成
