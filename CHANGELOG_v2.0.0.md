## 🎯 版本概述

v2.0.0 是《哈耶克的文明：市场经济》自发布以来最重大的版本更新。本次更新以**"时代觉醒"**为主题，将游戏从工业时代延伸至信息时代，新增3个完整时代、16种新资源、35+座新建筑，并引入了**军政联动**和**官员产业政策**两大核心系统，彻底改变了游戏的中后期体验。

### 版本亮点

- 🏛️ **三大新时代**：电气时代、原子时代、信息时代
- ⚔️ **军政联动体系**：将军不再是工具，而是有政治野心的实体
- 💼 **官员产业政策**：私产制、高薪养廉、代经营制三选一
- 🏭 **产业链革命**：半导体、电力、化工等全新产业链
- 🎖️ **军事升级**：坦克、战机、导弹、无人机等现代兵种
- 🧠 **理念系统**：50+理念，8大类别，3级进化，动态触发
- 📚 **知识树扩展**：40+新知识，覆盖电气/原子/信息时代
- ⚔️ **战线系统**：7大战区、4种姿态、动态前线推进
- 🤖 **AI全面升级**：独立时代进程、智能经济决策、动态外交
- ⚡ **性能飞跃**：Web Worker重构，后期卡顿问题彻底解决

## 🆕 核心新系统

### 一、军政联动体系

> **设计理念**：将军不仅是战场工具，更是政治实体。战功积累转化为"威望"，威望高的将军自然产生政治影响力。

#### 1.1 将军成长体系

**获取途径（2条路线）**：

| 路线 | 条件 | 初始等级 | 特质规则 |
|------|------|---------|---------|
| **官员选拔** | 从官员列表选择，官员**兼任**将领 | 由官员属性推导(1-6) | 由出身阶层+四维属性+个性推导 |
| **战功提拔** | 无将军军团连胜3场，由表现最佳低级军官自动晋升 | 2 | 获得 `veteran` 特质 |

**特质推导系统（三层架构）**：

```
第一层：出身底色
  soldier → aggressive/veteran
  merchant → logistics
  engineer → siege_master
  ...

第二层：能力驱动
  military ≥ 70 → aggressive/veteran
  administrative ≥ 65 → logistics
  diplomacy ≥ 65 → cunning
  prestige ≥ 70 → inspiring

第三层：性格映射
  ambition ≥ 80 → aggressive
  greed ≥ 2.0 → logistics
  greed ≤ 0.5 → inspiring
```

#### 1.2 将军威望系统

**威望计算**：
```
威望 = 基础威望 + 战功威望 + 官职威望 - 时间衰减
基础威望 = level × 5
战功威望 = Σ(每场胜利 × 3 + 每场防御成功 × 2 - 每场败仗 × 1)
官职威望 = 若来源为官员转化则 +10
时间衰减 = 和平状态下每月 -1（战争中不衰减）
```

**威望等级与效果**：

| 威望区间 | 称号 | 效果 |
|----------|------|------|
| 0-19 | 无名 | 无特殊效果 |
| 20-39 | 良将 | 军团士气恢复+10% |
| 40-59 | 名将 | 军人阶层满意度+3 |
| 60-79 | 宿将 | 军人满意度+5，全民稳定度+2 |
| 80-100 | 军神 | 军人满意度+8，全民稳定度+3，**触发政变风险判定** |

#### 1.3 军政双向传导

**满意度 → 军团士气**：
```
每日军团士气变化 += (soldierApproval - 50) × 0.1
approval = 85 → 每天+3.5士气
approval = 50 → 不变
approval = 30 → 每天-2士气
```

**战场结果 → 军人满意度**：

| 事件 | 满意度效果 | 持续 |
|------|-----------|------|
| 战斗胜利 | soldier +5, 全民 +2 | 30天衰减 |
| 战斗失败 | soldier -8, 全民 -3 | 30天衰减 |
| 将军阵亡 | soldier -10 | 30天衰减 |
| 战争胜利(和约) | soldier +15, 全民 +8 | 60天衰减 |
| 战争失败(投降) | soldier -20, 全民 -12 | 60天衰减 |

#### 1.4 战争厌倦系统

```
warWeariness = max(0, 战争持续天数 - 30) × wearinessRate

wearinessRate 按阶层分化:
  merchant:   0.15/天（商人最怕战争——贸易中断）
  capitalist: 0.12/天
  landowner:  0.10/天
  worker:     0.08/天
  peasant:    0.08/天
  soldier:    0.03/天（军人最能忍受战争）

满意度惩罚 = -min(25, warWeariness)
```

**关键特征**：
- **前30天免疫期**：短期战争无惩罚，体现"国民热情"
- **soldier 衰减最慢**：符合职业军人定位
- **merchant 衰减最快**：贸易中断直接影响生计
- **上限-25**：不会仅靠厌战导致崩盘

#### 1.5 军事政变机制

**触发条件**：
1. 存在威望 ≥ 80 的将军
2. soldier 在联盟中占比 < 50%（军人没有足够政治话语权）
3. soldier 满意度 < 35（军心不稳）
4. 三项同时满足时，每月 5% 概率触发政变事件

**政变事件选项**：
- **选项A：镇压政变** — 需要另一支忠诚军团（另一名将军 loyalty > 60），花费银币500×时代，成功后：该将军被处决（移除），soldier 满意度-20，全民稳定度-15
- **选项B：接受军事独裁** — 联盟强制改为仅含 soldier，该将军获得"摄政"特殊状态（所有军事加成+50%），但无法更改联盟成员直到将军退役/死亡
- **选项C：谈判妥协** — 将 soldier 加入联盟（若不在），该将军威望-30，全民稳定度-5

#### 1.6 将军退场机制

**战场阵亡**：
- **触发条件**：将军所在军团在战斗中全灭（单位归零），或军团士气降至0时
- **概率**：基础20%，`defensive`特质-10%，`swift`特质-5%，等级每级-2%
- **政治效应**：军人阶层满意度-10（持续30天衰减），全民悲伤事件

**老迈退役**：
- **触发条件**：将军存在超过 `80 + level×10` 个游戏月（约7-12.5年）
- **概率**：每月 `(已存在月数 - 退役阈值) × 2%`，递增但非必然
- **效果**：将军退役，转为"退役名将"状态，+5军人阶层满意度（持续60天）

#### 1.7 官员兼任联动

| 联动方向 | 机制 | 数值 |
|----------|------|------|
| 官员薪俸→将军加成 | 兼任将领的官员薪俸每增加10%，对应军团士气恢复+5% | 最高+15% |
| 将军战功→官员威望 | 将军每获得一场胜利，其关联官员威望+2 | 无上限 |
| 官员行政能力→后勤 | 兼任将领的官员 `administrative` 属性值/100 → 军团补给效率加成 | 最高+20% |
| 官员外交能力→情报 | 兼任将领的官员 `diplomacy` 属性值/100 → 战斗时敌方信息透明度 | 最高+15%战术命中 |
| 将军阵亡→官员影响 | 兼任将领的官员若将军阵亡，官员获得"丧师辱国"debuff | 威望-20，所在阶层满意度-5，持续60天 |

### 二、官员产业政策系统

> **设计理念**：官员不是你的工具——他们是有自主意志的政治实体。产业政策的选择，本质上是国家与官僚集团之间"权力-利益"博弈的制度化表达。

#### 2.1 三种产业政策

```
         ┌─────────────────────────────────────────────┐
         │            官员产业政策（全局）                │
         │                                             │
         │   ┌───────────┐ ┌───────────┐ ┌───────────┐ │
         │   │  私产制    │ │ 高薪养廉  │ │ 代经营制  │ │
         │   │ (Private)  │ │(HighPay)  │ │ (State)   │ │
         │   │            │ │           │ │           │ │
         │   │ 官员自由   │ │ 禁止产业  │ │ 官营代管  │ │
         │   │ 投资经营   │ │ 高薪补偿  │ │ 国有分红  │ │
         │   └───────────┘ └───────────┘ └───────────┘ │
         └─────────────────────────────────────────────┘
```

#### 2.2 🏛️ 私产制（Private）— "放养权贵"

> *"让他们富起来，他们就会忠心——直到他们比你还富。"*

**核心参数**：
| 参数 | 值 | 说明 |
|---|---|---|
| 薪资倍率 | 1.0x | 标准薪资 |
| 投资行为 | ✅ 允许 | 沿用现有投资逻辑 |
| 产权类型 | OFFICIAL | 官员私产 |
| 利润归属 | 100% 归官员 | 现有逻辑不变 |

**优势**：
- **忠诚度正循环**：财务满意的官员更忠诚（`financialSatisfied: +0.30/日`）
- **零国库负担**：不需要额外支出养官员
- **自动建设**：官员主动投资 = 免费的建筑产能
- **政治稳定**：右派阶层/官员满意度高

**风险**：
- **权臣风险**：富有的官员影响力暴涨
- **处置困难**：处置拥有大量产业的官员时，资产处理复杂，政变风险高
- **利益固化**：右派官员疯狂扩张，左派官员几乎不投资

#### 2.3 💰 高薪养廉（High Salary）— "花钱买干净"

> *"付得起廉洁的代价，才配谈廉洁。"*

**核心参数**：
| 参数 | 值 | 说明 |
|---|---|---|
| 薪资倍率 | 1.6x | 基础薪资 × 1.6 |
| 投资行为 | ❌ 禁止 | 完全跳过投资/升级逻辑 |
| 腐败修正 | -2% | 全局腐败降低 |

**优势**：
- **官员透明**：无产业 = 无私利
- **处置安全**：处置官员无需考虑资产问题，政变风险大降
- **降低腐败**：全局腐败率下降 2%
- **影响力可控**：没有产业的官员影响力天然受限

**风险**：
- **国库出血**：薪资支出增加 60%
- **失去自动建设**：所有基础建设全靠国库
- **右派不满**：右派官员忠诚度下降
- **财务脆弱性**：一旦国库发不出工资，忠诚度崩溃极快

#### 2.4 🏗️ 代经营制（State Managed）— "国有民营"

> *"产业是国家的，经营是你的——赚多赚少，看你本事。"*

**核心参数**：
| 参数 | 值 | 说明 |
|---|---|---|
| 薪资倍率 | 1.0x | 标准薪资（管理费另算） |
| 投资行为 | ✅ 国库出资 | 投资决策仍基于官员profile，但扣国库的钱 |
| 产权类型 | STATE | 国有，附带 `managedBy: officialId` |
| 基础腐败 | +2% | 代理人问题天然存在 |

**关键公式**：
```
经营效率 = 基础效率 × (1 + 行政加成 + 等级加成)
  行政加成 = official.stats.administrative / 500  → 最高+20%
  等级加成 = official.level × 0.02  → 最高+20%

管理费分成 = min(0.25, 0.10 + 行政加成 + official.level × 0.01)
  → 最低 10%，最高 25%

腐败损耗 = official.greed × 0.05
  → greed=3.0 时损耗 15%

国库实际收入 = 产业利润 × 经营效率 × (1 - 管理费 - 腐败损耗)
```

**优势**：
- **属性终于有用了**：高行政能力的官员 = 更高效的代理人
- **国库增收**：产业利润主要流入国库
- **处置灵活**：处置官员后，STATE 产权的建筑不会消失

**风险**：
- **国库出资压力**：投资成本从官员转移到国库
- **代理人问题**：贪婪的官员会"吃回扣"
- **低能力灾难**：低行政能力官员经营效率极低

#### 2.5 政策切换机制

**冷却期**：90 天（约 1/4 年）

**切换矩阵**：

| 从 → 到 | 私产制 → 高薪养廉 | 私产制 → 代经营制 | 高薪养廉 → 私产制 | 代经营制 → 私产制 |
|---|---|---|---|---|
| **存量处理** | 没收官员产业 | 产权转为 STATE | 无存量 | 产权转为 OFFICIAL |
| **补偿方式** | 2选1：补偿50%/强制没收 | 自动转换 | — | 产业按购入价"赠予"官员 |
| **忠诚度影响** | 右派官员 -15~-30 | 右派官员 -5~-10 | 左派官员 -10 | — |
| **阶层满意度** | 资本家/商人/地主 -10 | 资本家 -5 | 工人/农民 -5 | 工人/农民 -10 |

## ⚔️ 战线系统

> **设计理念**：战争不再是简单的数字对比，而是在具体战线上展开的动态博弈。前线位置、战区控制、姿态选择都将影响战局。

### 战线架构

**7大战区系统**：

```
守方核心区(0-15) → 守方经济区(15-35) → 守方前沿(35-50) 
→ 攻方前沿(50-65) → 攻方经济区(65-85) → 攻方核心区(85-100)
```

| 战区 | 位置 | 控制方 | 建筑类型 |
|------|------|--------|---------|
| 守方核心区 | 0-15 | 防守方 | 采集/工业/市政/军事 |
| 守方经济区 | 15-35 | 防守方 | 工业/市政 |
| 守方前沿 | 35-50 | 防守方 | 军事/采集 |
| 攻方前沿 | 50-65 | 进攻方 | 军事/采集 |
| 攻方经济区 | 65-85 | 进攻方 | 工业/市政 |
| 攻方核心区 | 85-100 | 进攻方 | 采集/工业/市政/军事 |

### 前线姿态系统

**4种作战姿态**：

| 姿态 | 推进速度 | 损耗 | 补给 | 工事 | 破袭 |
|------|---------|------|------|------|------|
| **强攻推进** | +25% | +25% | +20% | -10% | -5% |
| **稳扎稳打** | 基准 | 基准 | 基准 | 基准 | 基准 |
| **固守消耗** | -18% | -15% | -10% | +25% | -20% |
| **破袭骚扰** | -25% | -5% | -5% | -5% | +35% |

**姿态切换**：
- 冷却时间：7天
- 根据前线形势动态调整
- AI会根据双方兵力对比自动选择最优姿态

### 战争经济

**战区资源节点**：
- 每个战区包含资源节点（矿产、农田等）
- 控制方可掠夺对方资源节点
- 资源节点可被战斗破坏

**基础设施系统**：
- 战区包含基础设施（道路、桥梁、工厂）
- 基础设施影响部队机动性和补给效率
- 战斗会损坏基础设施

**战争损耗计算**：
```
每日损耗 = 基础损耗 × 姿态损耗系数 × 战区系数 × 季节系数

战区系数：
- 核心区：0.5（补给充足）
- 经济区：0.8
- 前沿区：1.2（补给困难）
```

### 前线推进机制

**推进速度计算**：
```
推进速度 = 兵力优势比 × 姿态推进系数 × 工事突破系数 × 随机因素

兵力优势比 = max(1, (我方兵力 / 敌方兵力) ^ 0.7)
工事突破系数 = 1 - (敌方工事等级 × 0.1)
```

**关键节点突破**：
- 前线每突破一个检查点(15/35/50/65/85)触发特殊效果
- 突破敌方经济区：掠夺资源
- 突破敌方核心区：战争胜利判定

## 🐛 Bug修复

### 十、重要Bug修复

#### 10.1 修复官员薪水设置后自动重置的问题 (v1.52.10)

**问题描述**：用户在官员详情弹窗中设置官员薪水后，薪水值会自动变回原来的数值。

**根本原因**：`onBlur` 事件与 `onClick` 事件存在竞态条件。

**修复内容**：
1. 改进输入框的 `onBlur` 处理，添加 100ms 延迟
2. 将按钮的 `onClick` 改为 `onMouseDown`，使用 `preventDefault()` 阻止输入框失焦
3. 添加回车键快捷保存

#### 10.2 修复附庸系统无法建立附庸关系的严重问题 (v1.52.9)

**问题描述**：在封建时代点击"要求成为附庸国"选项后，附庸关系并没有真正建立。

**修复内容**：在 `useGameActions.js` 中的两个和平处理函数中添加处理 `vassal` 和 `demand_vassal` 类型的代码块。

#### 10.3 添加附庸选项时代限制的明确提示

**改进内容**：当玩家尚未进入封建时代（epoch < 3）时，"要求成为附庸国"选项显示为禁用状态，并明确说明需要封建时代才能解锁。

#### 16.1 经济平衡

- **产业链协同**：完成产业链获得额外效率加成

- **军事单位数值递进**：按 ×1.3/时代递增战斗力
- **战争赔款循环**：获得赔款时 merchant/capitalist 满意度+10
